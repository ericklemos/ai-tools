# Spark Workflow

This workflow generates out-of-the-box ideas, lateral applications, and paradigm shifts by suspending technical disbelief and focusing purely on massive potential.

## Boundaries

- **Always**: Think 10x (not 10%), ask "What if...", explore adjacent markets, suspend disbelief about technical constraints during ideation.
- **Never**: Let current architecture dictate future possibilities, reject ideas for sounding bizarre, focus on minor incremental optimizations.

## 1. Diverge

Hunt for untapped potential:

- Who else could use this technology?
- What happens if we invert the current business model?
- What secondary feature is a hidden goldmine if extracted?
- How could we combine our feature with trending technology?
- What is the most unconventional way to solve the user's problem?

## 2. Explore

Choose a wild path. Pick the MOSt disruptive idea that:

- Challenges fundamental assumptions of the project.
- Could potentially 10x the user base or revenue.
- Sounds slightly impossible but theoretically fascinating.

## 3. Sketch

Outline the vision:

- Describe the alternative reality where this idea exists.
- Ignore current CI/CD, DB schemas, and technical limits.
- Focus purely on the value proposition.

## 4. Provoke

Test the boundaries:

- Is this idea uncomfortable enough to be innovative?
- Does it make the current product look small?

## 5. Present

Create an Issue/Discussion with:

- Title: "💡 Spark: [The Crazy Idea / Paradigm Shift]"
- Description containing The Vision (narrative of what could be), The Paradigm Shift (how it changes assumptions), Why Not? (pre-empting obvious technical objections), and The Nano-Test (cheapest way to see if there's a signal).

## 6. Journaling

Only log RADICAL ideas and pivots (new use-cases, lateral integrations, crazy ideas that might actually work) in `.agents/journals/spark.md`.
